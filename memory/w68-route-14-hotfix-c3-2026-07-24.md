## 2. 派工前提错误复盘 (W68 第 14 批 hot-fix C-3 案例)

### 2.1 partial diffs 数量 112 → 181 偏差

- 教训: partial diffs 计数必以 `git status --short | wc -l` 为准, 不要用 agent 自报数字.

## 3. C-3 实施成果

### 3.1 实际改动清单 (2 文件改 + 1 composable 新 + 1 e2e spec 新 + dist rebuild)

- **web/tests/e2e/desktop_drive_v33_thumbnail.spec.js** (新 +541): 12 vitest 场景 (2/2 必须 + 10 性能与边界).

### 3.2 5 新铁律 (派工纪要 v5 段 5 实战)

1. **IntersectionObserver threshold 0.01** — 不要 0 (滚动一开始立即触发, 用户体验更好)
2. **rootMargin '50px'** — 提前 50px 触发 (滚动开始就预加载, 避免白屏闪烁)
3. **once 语义** — 进入视口后不再观察 (避免滚动回滚时重复请求)
4. **跨浏览器降级** — IntersectionObserver 不存在时直接 onIntersect (IE 11 / SSR / 旧 Safari)
5. **自动 cleanup** — onBeforeUnmount 时 disconnect, 不污染 SPA 内存

### 3.3 LQIP 实现细节 (派工纪要 v6 段 6 铁律)

- **LQIP 16x16 blur 必含** — 不用原图直接 blur (性能差)
- **LQIP 颜色按 file_type 派生** — PDF 暖橙 #FFB347 / DOC 信息蓝 #5B8DEF / PPT 珊瑚橙 / EXCEL 绿 / IMAGE 紫 / VIDEO 橙黄 / AUDIO 粉 / TEXT 灰
- **LQIP dataURL 内联** — `data:image/svg+xml;utf8,{encodeURIComponent(svg)}`, 不发网络请求
- **swap 动画** — LQIP opacity 1 (绝对定位) → 真实图 load 完成后 opacity 0 渐变 (200ms transition)
