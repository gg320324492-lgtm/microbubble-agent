## 5. 五条新铁律

### 铁律 1：plans Status 闭环必同步（不能等下次派工才补）

- 主指挥在 merge plan 实施 commit 后，**必须**回头更新对应 plan 的 Status 段
- Status 段必须含真实 commit hash + 实施内容简述
- 占位符如 "待主指挥合并后填" 必须立即替换为真实 hash

### 铁律 2：完成 plans 必标 COMPLETED（不能停留在 NOT_IMPLEMENTED）

- plan 实施完 + commit merge 后，Status 段从 NOT_IMPLEMENTED 改为 COMPLETED
- COMPLETED 标记包含：commit hash + 实施内容 + 测试通过情况（如 "5/5 e2e PASS"）
- 不允许"实施了但 Status 段忘了改"现象累积

### 铁律 3：alembic 070 临时编号 必主指挥合并后改 076（串单链纪律）

- 并行派 2+ 个写 alembic migration 的 agent 时，临时 down_revision 接同一个上游
- 主指挥合并时按 alembic 链顺序改接续关系（B-1 改 076 接 B-2 改后的 075）
- 合并后立即 verify 只 1 个 head：`python -c "from alembic.config import Config; ..."`

### 铁律 4：6 留 W70+ 派工 backlog 必（不写 plan body 只在 memory 记录）

- 调研发现真未实施 plans 时，**不**立即改 plan body 改 NOT_IMPLEMENTED
- 在 memory（如本文件 §3 表格）记录 backlog，留 W70+ 派工
- 主指挥后续派工时直接读 memory backlog，避免重复调研

### 铁律 5：plans 调研必 run 真验证（不信 Status 段自报）

- 调研 plan 实施状态时必须 3 步并行：
  1. 读 plan Status 段
