## 5. 沉淀 5 条新铁律 (主拍决策 + 派工前提)

### 铁律 1: feature branch 已实施不算闭环 — 必须 merge 进 main 才算状态化

- **触发**: W68 第 10 批 B-1 + B-4 实施成功 (commit hash 在 git log 可查), 但 NOT in main HEAD
- **违反后果**: CLAUDE.md "W68 第 10 批 grand closure" 段误记为 "已实施", 后续派工会重做
- **纪律**: 派工前提必跑 `git merge-base --is-ancestor <commit> HEAD` 验证, 输出 true 才是真 in main

### 铁律 2: 派工前提 stat 验证 4 grep (派工纪要 v4 铁律 3 实战应用)

- **触发**: W68 第 12 批 D-1 派工纪要 v3 铁律 + W68 第 13 批 D-1 派工纪要 v4 铁律对 plans 调研场景明确, 但对 "feature branch 已实施但未 merge" 场景未明确
- **填补**: 任何派工前提调研任务必须跑 4 grep:
  ```bash
  grep -rn "<key>" --include="*.py" <core_paths>  # 应用层
  grep -rn "<key>" --include="*.py" scripts/        # 运维脚本
  git log --oneline --all | grep -iE "<key>"        # 历史 commit
  grep -rn "<key>" --include="*.yml" .github/        # CI 配置
  ```
- **补充 grep**:
  ```bash
  git merge-base --is-ancestor <commit> HEAD         # commit ancestor 验证
  ls <subdir> 2>&1                                  # 子目录存在性
  wc -l <key>.{jsonl,py,yml}                        # 文件大小 + 行数
  ```

### 铁律 3: save_to_kb 5 道防线含义在不同版本不同 (主拍必拍)

- **触发**: W5 v3.1 + W62 D2 现状 5 道防线 (分数 + 内容 + 意图 + 灰度 + 备份) ≠ 子 plan ② D8 调研 5 道防线 (dedup + 长度 + 拒答 + 敏感词 + 抽检)
- **违反后果**: W69 派工实施口径与 D8 调研不符, agent 必返工
- **纪律**: 任何 save_to_kb / 自动入库相关派工, 主拍必先选型 (选项 X 重写 / 选项 Y 保留), 派工 prompt 必明示

### 铁律 4: Celery auto_intake_rollback 必作为独立任务 (非 CLI 工具升级)

- **触发**: 现状 `scripts/auto_intake_rollback.py` 119 行 CLI 工具, **NOT Celery task**, **NOT beat schedule**
- **违反后果**: 7 天 rollback 漏跑 = KB 累积污染, 必须 cron 手动兜底
- **纪律**: W69 派工 B-3 必新建 `app/services/qa_bench_tasks.py:auto_intake_rollback_task` (Celery task) + 配置 `app/core/celery.py` beat schedule (cron `0 4 * * *`)

### 铁律 5: Dashboard MVP 视图选型主拍必填 (ECharts vs el-card)

- **触发**: W68 第 7 批 A-4 实施 4 ECharts 子图 (含 ECharts 部署) ≠ D8 调研 4 el-card 简单视图
- **违反后果**: W69 派工 B-5 改造 4 ECharts → 4 el-card = 大返工
- **纪律**: 主拍必填保留 ECharts / 改 4 el-card, 派工 prompt 必明示
