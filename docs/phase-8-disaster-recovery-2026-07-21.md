## 1️⃣ 本地备份现状评估

### 已就绪 (6 脚本)

| 脚本 | 行数 | 用途 | 频率 |
|---|---|---|---|
| `scripts/backup_db.sh` | 42 | `pg_dump` + gzip | 每天 02:00 cron (已确认 Jul 18-20 自动跑) |
| `scripts/backup_minio_daily.py` | 299 | MinIO bucket 全量 (urllib + S3 API, 零外部依赖) | 每天 03:30 (避开 DB 备份) |
| `scripts/restore_from_backup.py` | 506 | PR6-P10/P11+ 通用 restore CLI (3 步 scan/validate/apply --confirm) | 按需 |
| `scripts/backup_scheduler.bat` | — | Windows Task Scheduler 包装 | 自动 |
| `scripts/install-backup-scheduler.ps1` | — | Windows 计划任务一键部署 | 一次性 |
| `scripts/local-backup.ps1` | — | PowerShell 本地备份 (Windows 主用) | 按需 |

### 数据规模 (实测)

| 资源 | 大小 | 备份频率 | 保留期 |
|---|---|---|---|
| PostgreSQL DB | 4.6 MB / dump (gzip 后) | 每天 02:00 | 7 天 (默认) |
| MinIO bucket | 197 GB 累计 | 每天 03:30 | 30 天 |
| backups/ 目录 | 115 MB (16 文件) | — | 历史保留 |
| git repo | 736 MB (1958 commit) | 实时 | 永久 |
| app/ | 2.9 MB | git tracked | 永久 |
| web/dist/ | 4.7 MB | git tracked | 永久 |

### 评估结论

- ✅ **本地备份完整**: DB + MinIO + git 三层防御
- ✅ **恢复脚本就绪**: `restore_from_backup.py` 3 步范式, 支持 upsert + table 覆盖
- ✅ **自动调度**: Windows Task Scheduler + cron 双支持
- ⚠️ **单点风险**: 本地磁盘损坏 / 服务器物理故障 = 全部数据丢失
- ⚠️ **git 不够用**: git 只保源代码, 不保 DB 数据 + 用户上传文件 (MinIO)

**单点故障场景**:
1. 服务器物理损坏 (硬盘坏 / 主板烧): 全部数据丢失, 只能从 git 重新部署 (缺 DB + MinIO 数据)
2. 误删数据 (admin 操作失误): backup + git reflog 可恢复 (CLAUDE.md 7/02 教训)
3. 攻击 (勒索软件 / 数据被改): 本地备份也可能被污染, 需要异地冷备

## 2️⃣ Cloud 镜像 / 异地冷备方案 (3 选项)

### 数据规模估算

- DB 备份: 4.6 MB × 30 天 = 138 MB
- MinIO 备份: 197 GB × 1/7 (增量) ≈ 28 GB/月 = 84 GB (3 个月)
- 总: ~85 GB 标准存储

### 选项 A: 阿里云 OSS (推荐, 国内访问快)

| 项 | 详情 | 月成本估算 |
|---|---|---|
| 存储类型 | 标准存储 (Standard) | ¥0.12/GB/月 × 85GB = **¥10.20/月** |
| 跨区域冗余 | 阿里云跨 region 自动复制 (同 AZ 三副本 + 异地 region 1 副本) | ¥0.20/GB/月 (异地副本) × 85GB = **¥17/月** |
| 请求费用 | PUT 0.01 元/万次, GET 0.01 元/万次 | < ¥1/月 (每天 1 次 PUT) |
| 传输费用 | 内网免, 外网 ¥0.50/GB (仅灾难恢复时拉取) | < ¥1/月 |
| 加密 | SSE-KMS / SSE-OSS (AES-256) | 免费 |
| **月总成本** | | **¥27-30** |

**优势**:
- 国内访问快 (走内网, 50 MB/s+)
- 跨 region 冗余自动 (同 AZ 三副本 + 异地 region 1 副本)
- S3 兼容 API (rclone 直接对接)
- KMS 加密集成 (跟项目 MinIO 一致)

**劣势**:
- 需要阿里云账号 + AccessKey 管理
- 国内合规要求 (实名认证)

### 选项 B: 腾讯云 COS (备选, 国内访问快)

| 项 | 详情 | 月成本估算 |
|---|---|---|
| 存储类型 | 标准存储 | ¥0.13/GB/月 × 85GB = ¥11.05 |
| 跨区域冗余 | 多 AZ 复制 (3 副本, 单 region) | 含 |
| 异地冷备 (跨 region) | 跨 region 复制 | ¥0.18/GB/月 × 85GB = ¥15.30 |
| **月总成本** | | **¥26-30** |

**优势**: 跟阿里云类似, API 兼容性略差

**劣势**: 国内访问速度略慢于阿里云

### 选项 C: AWS S3 (海外, 合规友好)

| 项 | 详情 | 月成本估算 |
|---|---|---|
| 存储类型 | S3 Standard | $0.023/GB/月 × 85GB = $1.96 |
| 跨 region 冗余 | S3 Cross-Region Replication (CRR) | $0.023/GB/月 × 85GB = $1.96 |
| 请求费用 | PUT $0.005/千次 | < $1 |
| 传输费用 | 出网 $0.09/GB | < $1 (灾难恢复时) |
| **月总成本** | | **$5 ≈ ¥35** |

**优势**:
- 全球可用, 合规友好 (GDPR / HIPAA)
- 9 9.999999999% (11 个 9) 持久性
- 跨 region 复制成熟

**劣势**:
- 国内访问慢 (GFW 阻断, 走代理 1-5 MB/s)
- 海外账号需信用卡

### 方案对比

| 维度 | 阿里云 OSS | 腾讯云 COS | AWS S3 |
|---|---|---|---|
| 月成本 | ¥27-30 | ¥26-30 | $5 (¥35) |
| 国内速度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ (GFW) |
| 跨 region 冗余 | ✅ 自动 | ✅ 半自动 | ✅ 成熟 |
| 合规 | 国内实名 | 国内实名 | GDPR / HIPAA |
| API 兼容 | S3 | S3 | 原生 |
| 推荐场景 | 国内项目首选 | 腾讯生态 | 海外 / 合规 |
