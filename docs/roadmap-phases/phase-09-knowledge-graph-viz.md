# Phase 9: 课题组知识图谱可视化

> 周期：3-4 周 | 优先级：🟡 中

## 目标

将课题组的知识实体和关系以交互式图谱形式可视化，支持探索式学习和知识发现。

## 核心功能

### 9.1 实体关系网络
- 自动提取知识实体（人物、概念、方法、设备、材料）
- 识别实体间关系（使用、产生、依赖、相似）
- 构建知识图谱（节点 + 边）

### 9.2 交互式探索
- 力导向图布局（D3.js / ECharts）
- 节点拖拽、缩放、平移
- 点击节点查看详情
- 路径发现（两个实体间的关系路径）

### 9.3 智能推荐
- 基于当前浏览的知识推荐相关内容
- 发现隐藏的知识关联
- 知识空白识别

## 技术方案

### 前端
- D3.js 力导向图（已有 ECharts 力导向图基础）
- 节点样式：大小=重要度，颜色=类别
- 边样式：粗细=关联强度，虚线=弱关联

### 后端
- 实体提取：LLM + 规则引擎
- 关系抽取：共现分析 + 语义相似度
- 图谱存储：NetworkX + JSON（轻量级）

### 数据库
- `knowledge_entities` 表（已有）
- `knowledge_relations` 表（已有）
- 新增 `entity_embeddings` 向量索引

## 实现步骤

1. **Week 1**: 实体关系数据准备 + 图谱构建
2. **Week 2**: D3.js 可视化 + 交互
3. **Week 3**: 路径发现 + 推荐
4. **Week 4**: 性能优化 + 大图处理

## 验收标准

- [ ] 支持 1000+ 节点流畅渲染
- [ ] 节点可拖拽、缩放
- [ ] 点击节点显示详情
- [ ] 路径发现功能正常
- [ ] 推荐相关知识准确
