# Phase 12: 科研协作工作流

> 周期：4-6 周 | 优先级：🔴 高

## 目标

建立规范的科研协作流程，支持任务分配、进度追踪、评审流程。

## 核心功能

### 12.1 任务分配与追踪
- 任务创建、分配、状态更新
- 优先级、截止日期、依赖关系
- 进度可视化（甘特图/看板）

### 12.2 评审流程
- 实验方案评审
- 论文草稿评审
- 评审意见收集与处理

### 12.3 通知提醒
- 任务到期提醒
- 评审完成通知
- 进度更新推送

### 12.4 协作文档
- 共享文档编辑
- 评论与批注
- 版本管理

## 技术方案

### 前端
- 甘特图组件（dhtmlxGantt / 自研）
- 看板视图（拖拽排序）
- 实时通知（WebSocket）

### 后端
- 工作流引擎（状态机）
- 通知服务（已有企业微信/钉钉）
- 文档协作（OT 算法 / CRDT）

### 数据库
- `workflows` 表
- `workflow_steps` 表
- `reviews` 表

## 实现步骤

1. **Week 1-2**: 工作流引擎 + 状态机
2. **Week 3-4**: 评审流程 + 通知
3. **Week 5-6**: 甘特图 + 协作文档

## 验收标准

- [ ] 任务状态可流转
- [ ] 评审流程完整
- [ ] 通知及时送达
- [ ] 甘特图可交互
- [ ] 协作文档可用
