# Phase 8: 科研数据自动备份

> 周期：2-3 周 | 优先级：🔴 高

## 目标

建立完善的数据备份机制，确保科研数据安全，支持异地容灾和快速恢复。

## 核心功能

### 8.1 数据库定时备份
- PostgreSQL 全量备份（每日）
- 增量备份（每小时）
- WAL 归档（实时）
- 保留策略：7 天日备份 + 4 周周备份 + 12 月月备份

### 8.2 文件存储备份
- MinIO 对象存储备份
- 知识库附件备份
- 会议录音备份
- 声纹模型备份

### 8.3 异地容灾
- 备份文件自动上传到云存储（阿里云 OSS / AWS S3）
- 加密传输（AES-256）
- 跨区域冗余存储

### 8.4 恢复测试
- 自动恢复测试（每周）
- 恢复时间目标（RTO）< 1 小时
- 恢复点目标（RPO）< 1 小时

## 技术方案

### 备份工具
- pg_dump / pg_basebackup（数据库）
- rclone（文件同步）
- restic（加密备份）

### 存储后端
- 本地：/backup 目录
- 远程：阿里云 OSS / MinIO 异地实例

### 调度
- Celery Beat 定时任务
- 备份状态监控 + 告警

## 实现步骤

1. **Week 1**: 数据库备份脚本 + MinIO 备份
2. **Week 2**: 异地上传 + 加密 + 保留策略
3. **Week 3**: 恢复测试 + 监控告警

## 验收标准

- [ ] 每日自动备份成功
- [ ] 备份文件可正常恢复
- [ ] 异地备份上传成功
- [ ] 备份失败有告警通知
- [ ] 恢复测试通过
